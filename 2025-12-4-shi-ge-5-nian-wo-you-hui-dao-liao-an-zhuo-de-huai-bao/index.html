1 从诺基亚5230开始的手机折腾史

手机这东西，对我来说一直不只是通讯工具，而是某种“随身电脑”，承载着那段时间的生活方式。

1.1 高中：诺基亚 5230——第一次拥有“自己的设备”

真正意义上的第一台手机，是高中的诺基亚 5230。
那时候智能机刚刚普及，市面上都是那种九宫格键盘的手机，滑盖或者是26字母键盘已经算是比较潮流了。塞班系统还是主流系统之一，屏幕是电阻触控，操作有点费劲，但耐用得可以砸核桃。

5230 留给我最大的印象就是两个字：自由。

  • 可以自己折腾主题，改各种菜单图标；
  • 用数据线往里塞乱七八糟的 mp3、小游戏、电子书；
  • 宿舍里偷偷看小说、在被窝里用着2G网冲浪，那种“我与世界相连”的感觉非常强。

现在回头看，5230 大概就是我对“可玩性”和“可配置性”有了要求的起点。

1.2 大学：小米 4c、小米 Max 2、红米 Note 5——安卓时代的主力工具

上大学之后，正式进入安卓时代，主角依次是：小米 4c → 小米 Max 2 → 红米 Note 5，期间似乎还换过华为的平板。（更换太频繁不太记得了）

这么算来小米 4c 是我第一台真正意义上的安卓主力机。 那会儿的 MIUI 还处在一个很好玩的阶段：

  • 酷安还很纯粹，里面全是大佬，交流各种工具氛围很不错；
  • 随随便便root，刷第三方系统；
  • 接触到了小飞机，受到谷歌全家桶的热烈欢迎；
  • 各种第三方App、黑科技，只有你不会的没有你想不到的。

后来的 小米 Max 2屏幕大、电池大，拿在手里就有种“带着一块小平板走”的感觉，记得当初经常拿它来打炉石。

红米 Note 5 则是一个很标准的“水桶机”：性能够用、信号不错、拍照够看，没有明显短板。那几年基本就是安卓的一套完整生活体系：日常社交、支付、出行都在这台机器上完成；折腾自动化、备份、各种效率工具，也是在这几代安卓机上慢慢练出来的，什么冰箱、绿色守护、幸运破解器都是手到拈来。文件系统怎么管理、后台怎么放权、哪些 App 需要特殊权限，这些都已经变成肌肉记忆。

1.3 2020：iPhone 11——第一次“大幅换路”

2020 年换上 iPhone 11，算是一次比较彻底的“换路”。

一方面，是19年买了iPad（目前仍在役），感觉一些应用很不错只在苹果生态里有（没错就是Things和GoodNotes等笔记软件），有点想体验一下 iOS 的那种「不用管系统，只管用」的状态；另一方面，也确实是被大家说的稳定、省心、生态联动给种草了：

iOS 的动画和手势配合很顺滑，打开 App、在应用间切换的感觉确实很细腻；通知、权限、后台这些东西，都被系统管理得非常保守——省心，但也代表着另一种“被管着”。

在 iPhone 11 的这几年里，我慢慢适应了 iOS 的世界，习惯了那种“一切自动同步好、但你别管细节”的哲学。

1.4 2022：红米 Note 11 作为备用机——安卓从未真正离开

不过安卓并没有完全从我的生活里消失。 2022 年入手的红米 Note 11，从一开始定位就是一台备用机，主要理由有几点：

  • 手机号太多，一个手机不够（目前4张，绑定了一堆乱七八糟的账号也不敢注销）；
  • 苹果没有通话录音功能，还得是安卓；
  • 需要折腾自动化、远程管理、文件传输时，也经常会用到它（比如短信自动转发）；

可以说，从诺基亚 5230 到大学那几代小米，再到现在的备用机红米 Note 11，安卓这条线一直都在，只是这几年 iPhone 11 一直当着“名义上的主力机”。

2 为什么从iPhone回到了安卓

回想一下，这次从 iPhone 11 换回安卓，其实并不是一瞬间头脑发热，而是几个因素一路叠加，最后“啪”的一声把我推回了 Android 阵营。

2.1 拆换电池的“事故现场”，给了最后一击

直接导火索是——我自己动手换电池翻车了。

iPhone 11 用到后期，电池健康一路往下掉，其实早就不太抗用了。本着“勤俭持家 + 动手能力还行”的原则，我没去官方，也没去路边小店，选择了最危险的一条路：自己拆机换电池。

结果就是：

  • 拆的时候各种查视频、看教程，自以为步骤都对；
  • 真上手的时候，排线、卡扣、胶条，处处都是坑；
  • 重新装好那一刻，心里已经有点发虚：防水肯定是彻底告别了，细节上总觉得哪儿不太对。

虽然手机最后还能开机，但屏幕直接废了没反应，解锁不了也不能备份数据。这件事也算是给了我一个心理上的明示——差不多可以考虑换主力机了。

2.2 用了五年的老机：耗电快、有点卡、空间也不够

即便不算拆机阴影，一台 2020 年的 iPhone 11 用到今天，也确实到了力不从心的时候。电池就不用说了，换之前基本就是出门久一点必带充电宝；日常刷刷社交、看看视频都还行，但一旦后台多开几个 App，明显感觉“有点卡”。
最直观的痛点其实是——空间不够用。 照片、视频、聊天文件、各种应用缓存，一点点把存储吃光；清理过几轮之后发现：这台机器已经不太适合当主力机了。

2.3 被国内应用的广告喂了太多“屎”

这条其实是 iOS 和 Android 共同的现实，但我确实有点被国内 app 的广告环境磨到心态爆炸。无论你用的是 iPhone 还是安卓，只要日常生活离不开那几款高频国产 app：

  • 打开就是横幅广告、信息流推荐、全屏弹窗；
  • 有的应用甚至会用各种“伪通知”“伪系统弹窗”把广告包装得像系统提示；
  • 关闭按钮还贼难找，且升级后稍微摇一摇就跳转了；

“我在用一台很干净的系统，打开的却是一堆在疯狂给我喂屎的 app。”

既然我用苹果避免不了骚扰，难道我的大安卓还治不了你区区广告？
这时候，安卓那一套“你可以自己掌管一切”的思路，又开始变得有吸引力了。

2.4 听说国内 Root 环境日益严峻——想在还能折腾的时候再折腾一次

最后一个原因有点“技术宅自留地”的味道：听说这几年国内 Root 环境越来越严峻。

  • 各家厂商对 Bootloader 解锁和 Root 的限制越来越严格（参考小米）；
  • 有的机型根本不开放解锁，有的需要各种申请流程，甚至后续系统更新也会对 Root 用户不太友好；
  • 支付、安全类 app 对 Root 环境的检测也越来越敏感（据说装了Xposed框架后容易打不开各银行app）。

这些变化让我有一种“窗口期正在关闭”的危机感—— 如果再不趁现在换一台相对好折腾的安卓机，以后可能就真的只能当一个“纯用户”，而不是一个“会在底层翻箱倒柜的人”了。 我并不是非要天天刷机、魔改系统，但我希望手里至少有一台设备：

  • 可以自己决定要不要解锁、要不要开 Root；
  • 可以玩玩自动化、文件系统、NFC、网络工具这些更接近“随身电脑”的东西；
  • 而不是永远被锁在一个“你只能在我规划好的范围里折腾”的小盒子里。

综合这四点：拆机阴影 + 老机力不从心 + 国内 app 广告环境 + 想趁还能折腾时再折腾一次，最后就自然走到了一个结论：

是时候再给安卓一次“主力机”的机会了。

3 为什么选择了一加而不是小米

从一开始，我筛手机的思路就很明确，不是“谁拍照好、谁充电快”这种大众评测维度，而是更偏“工具控”的几条：

  • 能不能解Bootloader，过程麻不麻烦，毕竟我立刻就要用起来的；
  • 解锁之后 Root 环境是否稳定，有没有成熟教程和社区；
  • 日常体验要够顺滑，不卡壳、不拖泥带水；
  • 厂商别一上来就把各种权限锁死，让我连发挥空间都没有。

在这几条的交集中，一加自然就浮上来了。

3.1 选机标准：先看 Root，再看配置

大致排了一圈：

  • 有的品牌干脆不开放 Bootloader 解锁，直接淘汰；
  • 有的要各种“申请开发者资格”“审核通过”，流程又长又不透明；
  • 还有一些机型是理论上能解锁，但 Root 后各种安全 App 直接不给你活路。

相比之下，一加这边的口碑就简单直接得多——“还算比较友好”：

  • 官方还在提供 Bootloader 解锁通道（虽然步骤也变得比早年复杂了一点，要申请深度测试）；
  • 国内外论坛、Telegram、酷安上都有一堆详细教程和踩坑记录；
  • 各种 Magisk 模块、定制 ROM、内核调教都有人在维护。

3.2 一加的“气质”：系统够干净，折腾空间又大

硬件层面其实现在大家都卷得差不多了，真正让我更倾向一加的是两点：

  1. 系统不算太花哨

    • 虽然现在的一加也用上了 ColorOS 系列，但整体还是维持了一种“偏干净”的调性；
    • 预装软件不至于多到离谱，大部分能关的都能关，通知和权限的调教也比较直观；
    • UI 动画、操作逻辑很接近“轻度定制的原生安卓”，没有那种层层套娃的感觉。

  2. 可折腾的空间够大

    • GitHub / 各种论坛上，你能找到一堆专门针对一加的脚本和工具；
    • 从Root、系统精简、自动化到高级电源管理，都有人趟过坑；
    • 这意味着我可以省掉很多“自己摸黑试错”的时间，直接站在前人的踩坑记录上往前走。

换句话说，一加在“普通用户体验”和“极客折腾空间”之间踩得比较平衡： 你可以当一台正常手机用，也可以在某个周末，把它当开发板玩一玩。

3.3 一加的 Root 环境：目前还能“愉快玩耍”

  • 解锁流程：

    • 安装深度测试app，填写申请
    • 一到两天申请通过（我是第一天拿到手机立刻申请，第二天早上6点看已经通过了）

  • Root过程：

    • 我是无脑选择了一加全能工具箱进行root
    • 注意读使用说明，以及提前下载好驱动和对应系统版本的ROM包就行

  • Root 后的生态：

    • 银行、支付类 App 虽然越来越敏感，但目前还有一套可用的绕行方案（我刷的SukiSU Ultra）；
    • 日常使用中，不会出现“Root 之后变成一台不能干正事的玩具机”的尴尬局面；
    • 通过GKD/李跳跳，可以解决95%以上的应用广告骚扰问题。

这些综合起来，让我有一种很微妙的安全感： 我不是为了 Root 而 Root，而是知道在我需要的时候，这条路是通的。

4 核心配置

这次从一加开箱到解锁、获取 Root，再到装好常用模块、把系统调成我顺手的样子，整个流程其实挺长的：

  • 包括解锁 Bootloader、刷入第三方 recovery；
  • 用 SukiSU Ultra 获取 Root 权限；
  • 做一些基础安全配置和备份；
  • 再根据自己的需求选装若干模块和工具。

6 两套系统的优缺点总结

维度 安卓（以一加 + Root 视角） iOS（以 iPhone 11 视角）
系统稳定性 大版本偶尔有小 bug，更新前需要“等别人踩坑”；配置到位后总体稳定 整体非常稳，几乎不用管系统；升级心智负担小
个性化 & 外观 桌面、图标、字体、主题、息屏、手势全都能改，想多花多花 图标和布局自由度有限，主题系统偏保守
文件系统 & 传输 真·文件系统，本地/NAS/U 盘随便挂；LocalSend 等非常好用 文件抽象层比较重，各应用隔离，用户一般看不到
自动化 MacroDroid/Tasker + Root 能做到“手机级小服务器” 快捷指令在系统框架内很好用，但触发条件和权限边界更严格
权限 & 后台 可以精细控制自启动、电池优化、后台保活；也更容易自己搞乱 系统一刀切，后台和推送很克制；省心但可控性弱
Root / 高级玩法 还能比较顺利解锁 + Root，有成熟社区和模块生态 等于没有 Root 这条路；安全一致，但完全不能往底层折腾
隐私与安全策略 看厂商，默认比 iOS 放得更开；Root 后需要自己做好风险管理 系统默认更保守，权限管得严；对“小白用户”更友好
电池与续航 电池大 + 快充爽，续航基本靠堆料和调教 优化不错，但老机型掉电明显；电池不可自己瞎折腾（我就翻车过）
与其他设备联动 主要靠第三方（NAS、远程桌面、云同步等）自己搭 和 Mac/iPad/Apple Watch 生态联动顺滑，开箱即用
折腾成本 vs 收益 前期要花时间配好；一旦搞顺手，会非常贴合自己的使用习惯 几乎不用折腾；但很多“小强迫症需求”永远满足不了
广告环境体感 系统层面可用工具压制一大部分 系统本身干净，但 App喂shi能力极强，很难压制
